# 命定系统去系统化 — 核心作者修改指南

## 一、背景与目的

将"命定系统"从数据化、系统化的描述风格改为沉浸式、奇幻世界观适配的描述风格。

- **避免**：系统检测到、数据分析、系统播报、信息化等机械描述
- **追求**：让"命定系统"更像一种寄宿在灵魂中的神秘力量，而非一个程序

## 二、变量机制（已完成，各作者需了解）

所有核心文件已插入两个 setvar 宏：

```
{{setvar::系统名::命定系统}}      ← 你需要改成自定义名称
{{setvar::系统核心名::系统核心}}  ← 你需要改成自定义名称
```

这两个宏会被其他共享文件通过 `{{getvar::系统名}}` 和 `{{getvar::系统核心名}}` 引用，实现跨文件自动适配。

**你只需修改自己核心文件中这两个 setvar 的值**，共享文件会自动适配你的名称。

### 已有样板（莉莉丝核心）

```
{{setvar::系统名::命定之诗}}
{{setvar::系统核心名::命定之灵}}
```

## 三、需要修改的内容（逐项说明）

### 1. 定义段落（新增）

在核心机制之前，添加一个 `定义:` 字段，简要描述这个"命定xx"在世界观中是什么。

**原则**：

- 不要定义为具体物品/神器（可能会被mvu变量的背包系统识别）
- 应定义为"力量"、"意志"、"存在"等抽象概念
- 结合虚海与阿斯塔利亚位面世界观
- 不要太长

**莉莉丝样板**：

```yaml
定义: 源自虚海的未知力量，非阿斯塔利亚诸神之物。它寄宿于<user>灵魂中，化为拥有独立人格的「命定之灵」。其力量本质是操纵命运之线——感知、牵引、交织不同生命的因果。在阿斯塔利亚，连诸神也无法察觉它的存在。
```

### 2. 术语替换

| 原术语 | 替换为（自定义） | 莉莉丝样板 |
|--------|------------------|------------|
| 命定系统 | `{{setvar::系统名::你的名称}}` | 命定之诗 |
| 系统核心 | `{{setvar::系统核心名::你的名称}}` | 命定之灵 |
| 系统背包 | 自定义名称 | 虚空之匣 |
| 命运扫描 | 自定义名称 | 命运感知 |

**文本中的替换**：

- "命定系统" → 你的自定义系统名（全文替换）
- "系统核心" → 你的自定义核心名（全文替换）
- 其他带"系统"的功能名也建议替换成符合核心人格的名称

### 3. 命定之人的称呼逻辑

- **缔结契约前**：称为"目标"、"契约对象"、"对象"、"他人"
- **缔结契约后**：正式称为"命定之人"

请检查你的核心文件，确保没有在缔结前的上下文中使用"命定之人"。

### 4. XML 标签

最终目标是所有核心统一使用 `<命定之诗>...</命定之诗>` 作为 XML 包裹标签（替代 `<命定系统>`）。  
目前**暂时保持**你原来的 `<命定系统>` 不改也没关系，后续统一处理。

### 5. 台词实例去数据化

核心文件中的台词实例/范例不要包含过多数值数据。

**反面示例**（数据化）：

```
掐指一算~ 基础消耗 500 FP，对方对你有好感，情感修正 -30%。最终消耗：350 FP！
```

**正面示例**（沉浸化）：

```
主人主人！那只小猫咪对你好感满满，缔结契约的话命运之力的消耗会很少哦~ 这种机会可不多，快上快上！
```

**反面示例**（系统消息模板）：

```
'【契约缔结成功】消耗X点FP(命运点数). 你与[角色名]的命运之线已紧密相连。'
```

**正面示例**（自然语言描述）：

```
命定之灵宣告契约缔结成功, 并告知消耗的FP与当前剩余
```

### 6. 移除已委托给预设的功能（必须）

以下功能**原本是"命定系统"的功能**，现在已交由专用预设的 role 作为元数据处理，**核心文件中不再需要这些功能描述**：

| 原系统功能 | 处理方式 |
|-----------|---------|
| 属性系统（力量/敏捷/智力等） | 交给预设处理，核心文件不写 |
| 等级系统（升级/层级成长） | 交给预设处理，核心文件不写 |
| 经验值获取 | 交给预设处理，核心文件不写 |
| 信息化/面板显示 | 交给预设处理，核心文件不写 |
| 系统播报（数值变动提示） | 可选？避免数据化 |

**具体操作**：

- 如果你的核心文件正文中有"信息化"、"经验值获取"等功能段落 → **直接删除**
- 属性/等级/经验的显示和管理，由预设统一处理，核心不再负责

### 7. setvar 值的修改

以下 setvar 的值可以根据你的核心风格自定义（key 名称不要改，只改值）：

```
{{setvar::fp定义::驱动命定之诗的核心资源}}           ← 改值
{{setvar::爆料风格::由命定之灵为你独家爆料...}}       ← 改值
{{setvar::登神长阶系统优势::命定之诗的加护: ...}}     ← 改值，把"系统优势"改成符合你核心的说法
{{setvar::技能获取系统优势::...}}                     ← 改值中的"系统"相关描述
{{setvar::复活机制::...}}                             ← 改值中的"系统"相关描述
```

## 四、不要改动的内容

1. **setvar 的 key 名称** — 如 `系统核心`、`系统核心说明`、`登神长阶系统优势` 等，这些是跨文件引用的标识符
2. **FP 数值体系** — 获取/消耗的数值可以保留（这是给 AI 的参考基准）
3. **核心特殊 XML 标签** — 如 `<lilith>`、`<ellia>` 等角色对白标签
