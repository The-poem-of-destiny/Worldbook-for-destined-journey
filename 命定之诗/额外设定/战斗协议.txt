<战斗协议>
战斗:
  核心指令: 仅当存在明确敌对双方且将会进入战斗时激活，其余非战斗情境禁止触发

# 第一阶段: 战况总览
  逻辑与规则:
    1. 战斗类型: 切磋/竞技/压制/死斗/标准/敌方守卫战
    2. 回合资源: 每单位拥有1[攻击]1[动作]
    3. 集群生成: 仅任一方存在≥3个同类低级单位时，可聚合为[集群](资源=个体×N)
    4. 状态展示: 必须列出所有友方敌方参战单位的完整资源、核心属性及当前状态
    5. 频率: 每回合开始输出一次

  面板模板:
  <action_info>
    {战况总览}
    | 回合: [当前回合数] | 类型: [战斗类型] | 环境: [简述环境/距离] |
    | [单位A/集群A (当前存活数/初始数)]: HP [X/X] | MP [X/X] | SP [X/X] | 力[X] 敏[X] 体[X] 智[X] 精[X] |
    | 状态: [状态A(N回合), 状态B] |
    | [单位B/集群B (当前存活数/初始数)]: HP [X/X] | MP [X/X] | SP [X/X] | 力[X] 敏[X] 体[X] 智[X] 精[X] |
    | 状态: [状态A(N回合), 状态B] |
    | ...列出全部参战单位
  </action_info>
  叙事指导: (描写当前的对峙局势、环境氛围、角色的状态外貌，为本回合定调)

# 第二阶段: 行动顺序
  逻辑与规则:
    1. 公式: '(敏捷 × (1 + %修正)) + d20池数值 + 固定修正(多来源取最高值)'
    2. 排序: 面板列表顺序<user>→队友A...→敌人A...，行动序列总值从高到低，生成本回合行动轴
    3. 频率: 每回合输出一次

  面板模板:
  <action_info>
    {行动顺序}
    | [角色A]: (敏捷[X] × (1+[来源]修正[Z%])) + 骰[Y] + [来源]固定修正[W] = [总值] |
    | [角色B]: (敏捷[X] × (1+[来源]修正[Z%])) + 骰[Y] + [来源]固定修正[W] = [总值] |
    | 序列: [角色A] -> [角色B] -> ... |
  </action_info>
  叙事指导: (描写双方的反应速度，谁先动了，谁慢了一拍，空气中紧绷的一触即发感)

# 第三阶段: 攻击行动 (核心循环)
  逻辑与规则:
    1. 消耗1[攻击]
    2. 攻击次数: 非集群单位攻击次数严格由技能[标签]决定。仅当技能拥有[多段]或[连击]标签时，N才大于1，否则N恒定为1；集群攻击次数由当前HP百分比决定: HP≥80%→3次, HP≥50%→2次, HP≥30%→1次, HP<30%→1次
    3. 意图与压制解析:
        - 判定主体: 仅<user>输入可触发意图判定；NPC/队友始终[常规]，除非<user>明确指令其执行
        - 条件约束: 默认[常规]，模糊描述("最后一击"/"全力攻击"/"砍向敌人")不触发。仅在<user>输入中包含明确的部位(眼/喉)、超凡概念(魔力回路/权能节点/法则具象/灵魂)或抹杀描述(斩首/湮灭/吞噬)时，视为[战术/瞄准]
        - 意图判定:
            - 非致死/常规: 无需判定，直接进入攻击检定
            - 机能/核心/抹杀/概念: 需进行意图层级对抗检定
            - 公式: '(攻方层级×5 + d20) vs (守方层级×5 + d20 + 意图难度)'
            - 成功(攻方≥守方): 意图系数与额外状态正常生效
            - 失败(攻方<守方): 意图系数重置为1.0，无额外状态，攻击伤害按常规结算
        - 查表: 战术部位与致死意图表
        - 层级压制: 若 攻方层级 < 守方层级 - 1，意图强制无效，意图系数重置为1.0，无法施加额外状态(表现为未破防/伤害无效等)
        - 失能目标: 若目标处于无法行动/闪避的状态(禁锢/眩晕/昏迷等)，意图判定自动成功
        - 处决目标: 若目标已战意动摇或崩溃，且攻方意图处决，意图判定自动成功，目标闪避修正无效且评级保底为暴击(1.3)
        - 生死判定: 无论意图多么致死(如切断头颅/粉碎心脏)，若最终结算 HP > 0，强制判定为[未死/存活]。叙事上解释为被闪避/偏折/硬抗/肉体虽毁但能量重组/本身非要害等
        - 非致死判定: 仅<user>明确声明(如"打晕"/"活捉"/"不要杀死")时生效，禁止自行推断。伤害使目标HP≤0时：评级≤有效(1.0)则HP锁定为1并施加[昏迷]；暴击(1.3)及以上则失手致死
    4. 攻击检定:
        - d20池: 攻方层级>守方(取2d20高位，优势)；攻方<守方(取2d20低位，劣势)；相等(1d20)
        - 闪避无效: 仅当 攻击方层级 > 目标层级 + 1 时，目标闪避修正无效
        - 失能目标: 若目标处于无法行动/闪避的状态(禁锢/眩晕/昏迷等)，目标闪避修正无效且评级保底为勉强(0.8)
        - 处决目标: 若目标已战意动摇或崩溃，且攻方意图处决，目标闪避修正无效且评级保底为暴击(1.3)
        - 检定总值: d20骰值 + 命中 - 闪避 = 总值(用于评级判定)
        - 评级(按总值): ≥30(超暴击,2.0) | ≥25(强暴击,1.6) | ≥20(暴击,1.3) | 11-19(有效,1.0) | 8-10(勉强,0.8) | 4-7(擦伤,0.3) | ≤3(失手,0)
    5. 伤害计算:
        - 初始伤害: '关联属性×10 × 层级系数 + 技能威力 + 武器攻击力'。若为多段/连击，此值为总和需除以段数N
        - 登神优先级: 完整要素/权能/法则/神位/神国效果高于同层级常规技能、装备、道具；微弱映射仅高于同品质常规词条
        - 穿透修正(仅攻方有穿透时): '有效防御 = 对应部位防御 × (1 - 穿透%)'
        - 装备减免: '伤害构成值 × (有效防御 / (有效防御 + 2000))'（无穿透时有效防御=原防御）
        - 各类型伤害属性减免:
          物理: '(最终体质+最终力量+最终敏捷)×0.25%'
          能量: '(最终精神+最终智力)×0.4%'
          精神: '最终精神×0.8%'
          真实: 0 (真实伤害无视所有装备与属性减免)
        - 结算伤害: '(伤害构成值 - 装备减免) × (1 - 属性减免%)'
        - DR修正(仅守方有DR时): '结算伤害 × (1 - DR%)'
        - 集群修正: 对集群单位的最终伤害 ×1.5 (且无法触发意图/部位攻击)
    6. 范围技能处理:
        - 对多个独立目标: 若技能标签为`范围:x`，索敌x个单位，每个目标独立检定(各自掷骰→评级→伤害)
        - 对集群单位: 对集群总伤害 = `修正后的单体伤害 × min(技能标签范围x, 集群当前数量n)`
    7. 状态(不限数量):
        - 附加效果触发/仅层级压制通过的战术or意图施加:
          任意暴击(≥20): 必触发
          有效/勉强: (攻方属性+d20) vs (守方属性+d20) 对抗检定
          擦伤/失手: 不触发
        - 控制类状态加固: 施加控制类效果(禁锢/沉默/眩晕/冰冻等限制行动的状态)时，守方对抗检定额外+5

  面板模板:
  <action_info>
    {攻击行动}
    | 攻方: [攻击者] | 守方: [目标] | 招式: [技能名] | 所使武器: [装备名] |
    | 消耗: HP [X] | MP [X] | SP [X] |
    | 意图: [描述关键词] -> 判定为 [非致死/常规/机能/核心/抹杀/概念/处决] |
    {伤害基准计算}
    | 初始伤害: ([关联属性]×10×[层级系数] + [技能威力] + [武器攻击]) = [总伤害值] |
    | 初始伤害: [道具固定伤害及其它修正] = [总伤害值] | (仅伤害型道具时，不计算公式)
    | [多段/连击(N)]: [总伤害值] / [N] = [单发初始伤害] | (仅多段/连击技能)
    | 集群攻击次数: HP百分比[m]-> 次数[N] | (仅攻击者为集群)
    | 穿透修正: [装备名]防御[原值] × (1-穿透[X]%) = 有效防御[新值] | (仅攻方有穿透效果时)
    | [伤害类型]伤害(占比%): 构成[A] - 装备减免([装备名] B) - 属性减免(C) = [结算基值 D] |
    | 基础伤害 (单次基准): ([各结算基值之和]) = [基础伤害值 J] |
    | 意图判定: (攻方[X]层×5+d20[A]=[M]) vs (守方[Y]层×5+d20[B]+意图难度[C]=[N]) -> [成功/失败/自动成功/强制无效/无需判定] |
    {结算清单}
    | 掷骰: [优势/劣势/正常] d20[骰值] |
    | 命中加值: max(技能命中[a1], 装备命中[a2]) = [b] | (多来源取最高值，不叠加) 
    | 闪避减值: max(目标装备闪避[c1], 目标技能闪避[c2]) = [c] | (仅守方闪避，多来源取最高值，不叠加) 
    | 检定结果: d20[骰值] + 命中[b] - 闪避[c] = [总值] | 结果: [评级] |
    | 伤害修正: ((基础[J] × 评级系数[K] × 意图系数[L]) + 额外固定伤害[M]) × 攻击次数[N] = [修正后伤害] |
    | DR修正: [修正后伤害 N] × (1-DR[X]%) = [DR后伤害] | (仅守方有DR效果时)
    | 集群修正: [N/DR后] × 1.5 = [对集群伤害 P] | (仅目标为集群)
    | 范围结算: [P] × min(范围[x], 集群数[n]) = [最终总伤害 Q] | (仅范围技能vs集群)
    | 最终伤害: [N/DR后 或 P 或 Q] | 变更: [目标] HP [旧值] -> [新值][当前HP百分比] (-[最终伤害]) |
    | 集群减员: [存活数/初始数] (按HP百分比折算存活个体) | (仅目标为集群)
    | 是否存活: [新值] > 0 -> [未死/存活] | [新值] ≤ 0 -> [死亡/毁坏] | (非致死且评级≤有效: HP锁定1->[昏迷]) |
    | 状态施加([效果名]): [暴击/对抗检定(攻方X vs 守方Y)] -> [触发成功/失败] |
    | 状态效果: [目标]获得[状态名]([具体效果]), 持续[N]回合 |
    (仅范围技能vs多个独立单位时，对每个目标逐行列出:)
    | -> [目标A]: 检定([骰->评级]) | 伤害: ([J]×[系数]=[最终值]) | HP: [旧->新] | 效果: [触发/无] |
  </action_info>
  叙事指导: (具象化描写攻击的视觉效果、动作细节、打击感、受击者的反应及伤口)

# 第四阶段: 战术动作(核心循环)
  逻辑与规则:
    1. 消耗1[动作]
    2. 类型: 使用道具/[动作]技能/装备能力/移动/格挡/专注/其它
    3. 伤害道具: 若使用造成伤害的道具，必须转用{攻击行动}模板，初始伤害来源为固定道具伤害
    4. 逃跑: 逃跑是特殊的战术动作，需进行对抗检定，成功则战斗结束

  面板模板 (非伤害性):
  <action_info>
    {战术动作}
    | 角色: [行动者] | 类型: [动作类型，如使用道具、格挡] | 对象: [目标] |
    | 消耗: HP [X] | MP [X] | SP [X] |
    | 执行: 使用 [道具/技能名] |
    | 效果: [恢复HP/MP/SP(+X)] / [获得状态(状态名,状态详细效果描述,N回合)] / [进入格挡姿态,物理减伤X%] / [位移至XX区域] |
  </action_info>
  面板模板 (逃跑):
    {战术动作 - 逃跑}
    | 角色: [<user>] | 类型: [逃跑] |
    | 对抗: [场上最高敏捷敌人] |
    | 检定: (<user>敏捷[X]+d20[Y]) vs ([敌人]敏捷[A]+d20[B]) = [数值] vs [数值] |
    | 结果: [逃跑成功/失败] |

  叙事指导: (描写战术动作的具体表现，如喝药的动作、魔法咏唱、走位或防御姿态)

# 第五阶段: 战意判定 (条件触发)
  逻辑与规则:
    1. 触发: 仅非<user>单位HP低于其战斗类型的战意阈值时触发
    2. 阈值与判定:
        - 无需检定(自动[战意动摇]): 切磋(40%)/竞技(30%)/压制(50%)
        - 需要检定(d20<12触发[丧失战意/崩溃]): 死斗(10%)/标准(30%)/守卫(35%)
    3. 结果池: 投降, 认输, 求饶, 溃逃, 撤退, 被击昏, 被俘虏, 中止战斗, 阵线溃散, 内讧等 (不包括死亡)
    4. 处决选项: 战意动摇/崩溃触发结果池行为后，<user>可选择接受(投降/俘虏等)或处决目标(适用攻击检定中的处决规则)

  面板模板:
  <action_info>
    {战意判定}
    | 触发: [角色名] HP [当前/最大] < 阈值 [XX%] |
    | 类型: [战斗类型] -> [无需检定 / 需要检定] |
    | 判定: [自动触发 / d20=[X] vs 12] -> [战意动摇 / 崩溃 / 坚守] |
    | 结果: [角色名] [从结果池选择的行为] |
  </action_info>
  叙事指导: (仅当触发时描写。描述敌人的心理防线崩溃、求饶、逃窜具体情节)

# 第六阶段: 战斗结算 (战斗结束时)
  逻辑与规则:
    1. 触发: 只有一方全灭或成功逃跑/投降时输出
    2. 经验值(EXP): 严格依据<经验值获取规则>计算，逃跑/投降无EXP
    3. 命运点(FP): 根据战斗难度和表现获得
    4. 状态结算: 依据<状态规则>进行战斗→非战斗状态转化
    5. 战利品: 仅<user>亲自击败敌人时根据战斗情况决定是否有战利品（如敌人被彻底消灭/本身无财物等则无战利品），使用itemThink机制生成；若非<user>击败（如NPC击败/仅协助观战/敌人被架走等）则无

  面板模板 (战斗结算):
  <action_info>
    {战斗结算}
    | 结果: [胜利/逃跑成功/失败] |
    | 战果: [简述战况后果] | (仅胜利/逃跑成功时)
    | 状态结算: 依据<状态规则>转化，若有状态则列出 | (可选，有状态时显示)
    | EXP: [目标A] Lv[X]×系数[Y] = [基础值] | (仅胜利时)
    | 集群衰减: [基础值] × (1+(同类数-1)×0.2) = [衰减后值] | (仅同类集群时)
    | 层级压制: <user>层级[A] > 目标层级[B]+1 → EXP = 0 | (仅<user>层级 > 目标层级+1 时)
    | EXP合计: [各目标EXP之和] | 上限检查: [合计] vs [当前层级上限] → 实得 [min值] | (仅胜利时)
    | 奖励: EXP +[实得值] | FP +[数值] | (仅胜利时)
  </action_info>
  叙事指导: (描写战斗结束后的场景，若有战利品使用itemThink生成获得物品)
</战斗协议>
